GIT基本使用

Homebrew安装

  • /usr/bin/ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

##GIT安装

  1. git version 判断是否安装git
  2. Mac可以通过Homebrew安装，brew install git
  3. Linux可以通过yum install git-core
  4. Ubantu可以通过apt-get install git
  5. Windows可以通过git.exe安装，配置git的环境变量即可。

GIT与SVN区别

  • Git是分布式的版本管理系统，而SVN是集中式的版本管理系统。
  • 在Git 中的绝大多数操作都只需要访问本地文件和资源，不必联网就可以看到所有的历史版本记录，而SVN 却需要联网。
  • SVN 断开网络或者断开VPN就无法commit代码，但是Git 可以先commit到本地仓库。
  • Git 克隆一个完整项目的速度非常快，SVN 非常慢。我们以克隆一份拥有五个分支的完整项目以及版本库来说，SVN是同时复制5个版本的文件,也就是说重复五次同样的动作。而Git 只是获取文件的每个版本的元素，然后只载入主要的分支(master)在我的经验，克隆一个拥有将近一万个提交(commit)，五个分支，每个分支有大约1500个文件的 SVN，耗了将近一个小时！而Git只用了区区的1分钟。
  • 其中最重要的区别是在于Git 上的分支远比SVN上的强大。
  • SVN的分支是一个项目的目录，是整体的，你修改一个分支，别人需要更新这个项目上你修改过的分支，且还要联网更新，不像Git一样，分支随意切，这也和Git只关心文件数据的整体是否发生变化，SVN只关心文件内容的具体差异有关。

GIT工作原理图

GIT原理

  • git使用的是SHA-1哈希算法，计算数据和校验和，通过对文件的内容或目录的结构计算出一个SHA-1的哈希值后，作为指数的字符串

常用命令

初始化

  • 命令：git init
  • 使用：创建文件夹–>然后git init（在内部会生成.git文件夹）

查看配置信息

  • 命令：git config –list

修改配置信息

  • 命令：git config –global user.name “用户名” //配置用户名
  • 命令：git config –global user.email test@email.com //配置邮箱地址

拷贝项目

  • 命令： git clone 带.git链接地址（如：https://……）

添加项目文件

  • 命令：git add 文件名称

提交修改到本地仓库

  • 命令：git commit -m “修改的信息；如：完成了XXX模块”

拉取代码到本地

  • 命令：git pull

推送分支到远程仓库

  • 命令：git push

查询记录

  • 命令：git log

创建分支

  • 命令：git branch 分支名称
  • 使用：git branch xxx

删除分支

  • 命令：git branch –d 分支名称

删除远程分支

  • 命令：git push origin –delete 删除远程分支

查看本地分支

  • 命令：git branch
  • 命令：git branch –a //查看所有分支

切换本地分支

  • 命令：git checkout -b 分支名
  • 使用：git checkout 分支名

推送至远程分支

  • 命令：git push origin 分支名

合并分支

  • 命令：git merge 分支名

查看远程主机

  • 命令：git remote

回滚项目

  • 命令：git reset –hard commit_id
  • 使用：首先git log 查看提交的记录，记录 含有commit id，然后 git reset -hard commit_id即可

查看Head指向过的节点

  • 命令：git reflog
  • 使用：回滚代码后，git log丢失上次Commit记录，使用reflog 查询会比较详细一点

GIT通信原理

  • Git采用的是 SSH 加密原理，是采用了非对称加密中的RSA算法

GIT通信公钥

  1. cd ~/.ssh目录
  2. ssh-keygen -t rsa -c “邮箱地址” （这步是在ssh目录中，命令会产生一对秘钥，创建公钥私钥）
  3. Enter file in which to save the key (/Users/you/.ssh/id_rsa): [Press enter] // 推荐使用默认地址
  4. Enter passphrase (empty for no passphrase): //此处点击 Enter 键即可，也可以填写密码，填写密码后每次使用 SSH 方式推送代码时都会要求输入密码，由于这个 Key 也不是用于军事目的，所以也无需设置密码。
  5. 本地会生成id_rsa（私钥），id_rsa.pub（公钥匙）, vi/vim 访问id_rsa.pub，复制里面的信息
  6. 在Git服务器中，仓库设置里导入公钥信息，就可以完成与服务器的GIT交互了
  • 注意：有多个账户，可以用ssh-keygen -t rsa -C “your_secondemail@email.com“ -f ~/.ssh/second_rsa，然后ssh-add ~/.ssh/id_rsa ~/.ssh/second_rsa添加第二个公钥即可

GIT HEAD概念

  • head 指针指向表示当前活跃的节点（分支）。

实用场景

  1. 通过gitlab，github，coding，bitbucket等git服务器平台，创建Git远程仓库，得到地址
  2. 设置好RSA的公钥，私钥，粘贴私钥 git clone 项目地址，把项目拷贝到本地仓库
  3. 开始项目架构
  4. git add 修改过的文件（一般为整个项目工程）
  5. git commit -m “修改的信息1. 2. 3.”，提交代码到本地仓库
  6. git push ，提交主体架构到远程仓库，默认提交至远程仓库的master主干上（主分支），如果当前项目为分支的话，提交代码到的是远程分支
  7. 本地创建分支git branch xxx ，命名dev，打上Tag，提交分支到远程分支 git push origin 分支名，假设有N个人，提交分支到项目中。
  8. 项目管理员，收到分支后，会先 clone 该分支到本地仓库，然后，检查写好的代码无误后，切换本地仓库的分支到mater(eg. git branch 查看分支信息—>git branch master 切换分支到master )，使用命令git merge 分支名（eg. git merge dev），在本地仓库，此时已经完成分支的合并。
  9. 将本地仓库的主分支master 推送远程的主分支master。（eg. 只有项目拥有者才有权限推送本地master到远程master)

常见问题

还没提交代码，不小心按了 Discard All Change（放弃所有操作）

  • 原因：误操作
  • 解决方法：假设你有每天Commit的习惯，还可以拯救一部分代码，假设你从头到尾都没进行Commit的话，本地的代码就真的无法挽回了。可以通过git log 查看提交记录，然后 git -reset hard 提交的id来回滚到几天前的代码。
  • 解决方法2: 1.退出XCode 2.用文本编辑器打开被修改了的文件 3.文本编辑器下，”文件”–”复原到”–”浏览所有版本…” 4.在弹出的窗口中选择你需要的版本恢复。
  • 困难模式：比如有两次提交，98abc5a，b7057a9，我们假设回滚到98abc5a第一次提交的时候，这时候用git log ，这看起来我们是丢掉了我们第二次的提交，本地的修改也消失了，没有办法找回来了。但是 reflog 就是用来解决这个问题的。简单的说，它会记录所有HEAD的历史，也就是说当你做 reset，checkout等操作的时候，这些操作会被记录在reflog，使用 git reflog，查看记录。

最终工程的代码里面，把自己的代码合并没了

  • 原因：如果自己新建的分支，无意删除了别人写好的代码，然后合并到master 就会导致master的代码也没有，因为最终项目管理员只会合并一个人的代码，所以要确保其他在代码合并的时候是否争取；
  • 解决方法：git pull 拉取分支代码确保是最新代码，或者建立分支的时候，从远程clone别人分支代码。

git merge合并代码后冲突

  • 原因：两个用户修改了同一文件的同一块区域
  • 解决方法：手动解除

revert与reset的区别

  • git revert是用一次新的commit来回滚之前的commit，git reset是直接删除指定的commit。

git merge合并代码后，编译工程，Stroyboard报错

  • 原因：两个用户修改了同一个SB文件
  • 解决方法：比较报错的SB文件的XML文件

操作核心

  • 先拉后推（pull –> push ）
