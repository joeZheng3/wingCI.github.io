加密方式对应的算法

方式 算法
对称加密 DES，AES
非对称加密 RSA
单向加密 Base64，MD5，SHA，HMAC

可逆/不可以加密

可逆

  • DES，AES，RSA，Base64

不可逆

  • MD5，SHA，HMAC

对称加密与非对称加密区别

  • 对称加密：服务端和客户端都是用同一个秘钥
  • 非对称加密：使用了一对秘钥，公钥与私钥，用公钥加密，用私钥解密

AES算法

  • 128位加密
  • 192位加密
  • 256位加密

RSA算法

  • 基于大数的因式分解的数学难题
  • RSA算法：由1978年提出，MIT学生，R，A，L共同提出

DES与AES的区别

  • DES以64位为分组数据进行加密（一般用3重des提高安全级别）
  • AES支持可变分组长度，分组长度可设定为32 比特的任意倍数，最小值为128 比特，最大值为256 比特。

RSA算法口诀

  • 公钥加密的数据，必须使用私钥才可以解密，同样私钥加密的数据，必须通过公钥进行解密

常见应用场景

  • 苹果的App Store采用的就是AES
  • Git 采用的SSH加密原理，是基于RSA非对称加密

实际项目采用的加密

  • （ 参数的value根据key降序/升序 + salt/key ） md5加密 得到 参数
  • 提高安全级别又想可逆：通常采用Base64+AES
  • 一般用3重des提高安全级别
