服务器基本操作

发表于 2017-07-18
字数统计: 1,601 | 阅读时长 ≈ 7

业务场景

  • 查找相关文件
  • 安装服务系统，Linux，ubuntu等
  • 上传后台或者网页代码到服务器上
  • 在服务器上安装解析网页的软件（nginx，Apache）
  • 在服务器上安装管理软件（Docker，Jenkins，gitlab，禅道）
  • 在服务器上安装监控软件 （Zabbix，Nagiox）
  • 搭建LAMP，LNMP等软件运行环境
  • 查看服务器软件运行日志
  • shadowsocks的搭建，构建VPN环境
  • 做监控，负载均衡，分布式，集群等大型的服务器架构

名词解释

  • VPS（Virtual Private Server 虚拟专用服务器）技术：将一部服务器分割成多个虚拟专享服务器的优质服务。就是我们经常买的云服务器，其实就是VPS的应用，买的只是某个服务器的某一部分。
  • 虚拟化：是一种资源管理技术，是将计算机的各种实体资源，如服务器、网络、内存及存储等，予以抽象、转换后呈现出来，打破实体结构间的不可切割的障碍，使用户可以比原本的组态更好的方式来应用这些资源，如：虚拟内存，虚拟机，Docker容器

服务器分区

  • 注意区分和windows的硬盘分区不一样，windows分(c,d,e盘)，linux只有一个盘下面一堆目录，每个目录都可以设置严格的权限chome

Linux系列常见目录

  • / :根目录
  • /bin :执行二进制文件的目录
  • /dev：硬件设备目录 比如：/dev/ram[0-15]：内存
  • /etc –> /init.d /sysconfig （一般软件安装完后都会在这个目录创建自己的目录 比如: /shadowsocks/config.json /nginx/nginx.conf） :环境目录
  • /home :用户界面
  • /mnt :外围设备目录U盘，移动硬盘之类的
  • /var –> /log /lib /spool /run /www (一般软件安装完后，都会在这生成一个日志，比如：shadowsocks.log，nginx.log ) :临时目录，一般存放日志之类的
  • /usr –>/bin /local /share :应用程序存放的目录
  • /boot :放置linux系统启动时用到的一些文件
  • /lib :系统使用的函数库的目录

应用流程

1.为Nginx配置域名

  • 购买万网的域名（ps.万网已经被阿里云收购）
  • 购买阿里云，亚马逊云，腾讯云，挑选对应的带宽，内存，对应的VPS，开启实例（省去自己购买服务器，配置服务器，安装系统的过程），启动服务器，获取自己服务器ip。
  • ssh登录服务器
  • 在自己服务器上安装Nginx或者Apacher
Ubuntu：
sudo apt-get install nginx //安装
sudo service nginx start //开启
sudo service nginx stop //关闭
sudo nginx -s reload //重启
或：
/etc/init.d/nginx //运行nginx
ps -ef | grep nginx //查询nginx进程
kill -QUIT 主进程号 //停止Nginx

Centos：
rpm -Uvh http://nginx.org/packages/centos/7/noarch/RPMS/nginx-release-centos-7-0.el7.ngx.noarch.rpm //挑选想要安装的源
yum install nginx  //安装nginx
service nginx start  //启动nginx
nginx -v //查看nginx版本
curl -i localhost //测试本地nginx

安装好的文件位置：
/usr/sbin/nginx：主程序
/etc/nginx：存放配置文件 
/usr/share/nginx：存放静态文件
/var/log/nginx：存放日志

思想：安装源，使用下载工具 apt-get / yum 下载，安装完后，启动服务 关闭服务
  • 设置nginx绑定域名

Nginx与Apache区别

  • nginx适合做静态，简单，效率高
  • apache适合做动态，稳定，功能强
  • Apache处理动态有优势
  • nginx并发性比较好，CPU内存占用少
