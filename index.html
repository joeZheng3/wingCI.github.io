Clarles常见使用

发表于 2016-05-19

场景

  • 进行App测试或定位线上问题时，经常会遇到抓取HTTP，HTTPS数据包的需求

抓取HTTP

  1. 下载Charles
  2. 替换JAR包破解
  3. 设置Proxy，Proxy Settings设置端口8888
  4. ifconfig查看本地wifi ip （注意：要和手机wifi同一个局域网）
  5. 设置ip，端口，进行抓包
  6. 电脑客户端界面Allow，即可抓取Http的包

抓取HTTPS

  • 设置Proxy，选择SSL Proxying Settings
  • Add 参数：Host–> * ，Port–> 443 这样就可以抓取443端口类型的接口
  • 设置Help –> Proxy Setting –> Download Moble Document
  • Configure your device to use Charles as its HTTP proxy on 192.168.3.126:8888, then browse to chls.pro/ssl to download and install the certificate. –> 手机安装Clarles描述文件
  • 安装描述文件

常见操作

  1. 查看请求Pram，Response，Request，Header
  2. 改写URL，伪装请求
  3. 设置网速限制，低网/无网进行请求
  4. 取消录制

库，框架，架构，平台，设计模式的思考

发表于 2015-05-17

库与框架的区别

  • 库的英语为Library（Lib），框架的英语为Framework
  • 类库/函数库：开发者在使用库的时候，只需要使用库的一部分分类式函数，然后继续实现自己的功能
  • 框架：是为了解决一个（一类）问题而开发的产品，框架用户一般只需要使用框架提供的类或函数，即可实现全部功能，可以说框架是库的升级版

形象比喻

  • 框架相当于一部组装好的电脑
  • 库就如自己的组装的电脑。库为我们提供了很多部件，我们需要自己组装，如果某个部分未提供，我们也可以自己做

彼此之前的区别

  • 库：工具箱
  • 框架：一套解决方案，重写代码就能够实现全部的功能
  • 架构：是高度抽象的需求（相当于房子的房型图）

设计模式与架构的区别

  • 设计模式 ：思想
  • 架构：方案，包含是设计模式和已存的框架与实际的业务需求
  • 框架：就是软件成果，能实现代码的重用，帮你实现一个类型的业务需求
